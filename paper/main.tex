%% ABSTRACT

\begin{abstract}
Resource-constrained Internet of Things (IoT) devices face an inherent
tension between post-quantum cryptographic security and energy
feasibility. While Ring-Learning-With-Errors (Ring-LWE) lattice
authentication and Quasi-Cyclic Low-Density Parity-Check (QC-LDPC)
code-based hybrid encryption provide quantum-resistant security, their
per-packet application imposes prohibitive computational, bandwidth, and
energy costs on Class-1 devices. This paper presents SAKE-IoT (Session-Amortized Key Establishment for IoT), a four-phase protocol that
amortizes the heavy post-quantum epoch-initiation cost over an entire
session of lightweight data packets. SAKE-IoT extends the Ring-LWE
ring signature authentication and QC-LDPC key encapsulation of Kumari
et al. with a 1-RTT piggybacked handshake, HKDF-SHA256 session-key
derivation, AES-256-GCM authenticated encryption per NIST SP 800-38D,
and epoch-bounded forward secrecy via secure memory zeroization. We
provide three formal security proofs (IND-CCA2, replay resistance,
Epoch-Bounded Forward Secrecy) and validate the protocol on two
orthogonal platforms: MATLAB for theoretical metric simulation and
Contiki-NG/Cooja for physical IoT network emulation over IEEE 802.15.4
with 6LoWPAN/RPL/CSMA. Network simulation results demonstrate a 99.7\%
end-to-end latency reduction in the data phase, a 45.1\% per-packet
bandwidth saving, and a 94.7\% session-level bandwidth saving at N=20,
with 100\% packet delivery ratio and two verified epoch renewal cycles.
\end{abstract}

%% I. INTRODUCTION

\section{Introduction}
\label{sec:intro}

The Internet of Things (IoT) connects billions of resource-constrained
devices across critical infrastructure---smart homes, healthcare
wearables, industrial control systems, and intelligent transportation
networks. These devices collect, process, and transmit sensitive
real-time data over wireless channels, making strong cryptographic
authentication essential. However, the anticipated advent of
large-scale quantum computers threatens all classical public-key
infrastructure based on RSA and Elliptic Curve Cryptography (ECC)
\cite{nist_pqc}.

Post-quantum cryptographic (PQC) schemes---primarily lattice-based,
code-based, and hash-based algorithms---resist quantum attacks.
Lattice methods such as Ring-LWE offer the best efficiency profile for
IoT hardware \cite{lyubashevsky2013}. However, applying Ring-LWE
operations \emph{to every data packet} imposes severe cost on RFC~7228
Class-1 constrained devices (8~KB RAM, 100~KB Flash, 8~MHz CPU)
\cite{rfc7228}:

\begin{itemize}
    \item \textbf{Authentication delay:} A single Ring-LWE + QC-LDPC
    handshake costs 7.37~ms of pure computation on Intel Core~i5
    hardware \cite{kumari2022}; the equivalent on an 8~MHz MSP430 is
    orders of magnitude slower.
    \item \textbf{Communication overhead:} One authentication message
    with ring signature ($N{=}3$, $n{=}512$) spans 10,317~bytes---162
    IEEE~802.15.4 fragments---for a single handshake.
    \item \textbf{Energy cost:} Repeated key encapsulation per packet
    drains battery capacity 30--33$\times$ faster than symmetric-only
    alternatives \cite{kumari2022}.
\end{itemize}

\subsection{The Amortization Gap}

Kumari et al.~\cite{kumari2022} introduced a powerful hybrid scheme
combining Ring-LWE ring signature authentication (LR-IoTA) with a
QC-LDPC code-based Key Encapsulation Process (KEP). Their scheme
provides quantum-resistant mutual authentication and identity privacy.
However, the data-sharing phase requires per-packet execution of the
full KEP, including QC-LDPC syndrome computation and AES session-key
derivation, on every transmitted message.

We identify this as the \emph{Amortization Gap}: the expensive
post-quantum epoch initiation should happen \emph{once per session},
with lightweight authenticated encryption serving all subsequent
packets. Inspired by TLS~1.3 session resumption \cite{rfc8446}, we
design SAKE-IoT to close this gap.

\subsection{Contributions}

This paper makes the following contributions:

\begin{enumerate}
    \item \textbf{SAKE-IoT Protocol:} A four-phase protocol extending
    Kumari et al., comprising a 1-RTT piggybacked post-quantum
    handshake, HKDF-SHA256 session key extraction, AES-256-GCM
    amortized data transmission, and epoch-bounded renewal with secure
    memory zeroization.

    \item \textbf{Three Formal Security Proofs:} IND-CCA2 security of
    the AEAD construction (Theorem~\ref{thm:indcca2}), deterministic
    replay attack resistance (Theorem~\ref{thm:replay}), and
    Epoch-Bounded Forward Secrecy via secure zeroization
    (Theorem~\ref{thm:ebfs}).

    \item \textbf{Dual-Platform Validation:} MATLAB theoretical
    simulation confirms 99.1\% computation reduction, 45.1\%
    bandwidth saving, and 33$\times$ clock-cycle reduction; Contiki-NG
    network emulation over IEEE~802.15.4 confirms 99.7\% E2E latency
    reduction, 100\% PDR, and two verified epoch renewal cycles---the
    first physical IoT network validation of a post-quantum session
    amortization protocol.
\end{enumerate}

\section{Related Work}
\label{sec:related}

\subsection{Lattice-Based Authentication for IoT}

Ring-LWE-based schemes offer favorable computational profiles for IoT
authentication. Li et al.~\cite{li2020} proposed lattice-based PAKE
using Smooth Projective Hash Functions. Wang et al.~\cite{wang2019}
introduced two-factor Ring-LWE authentication. RLizard
\cite{lee2019} demonstrated efficient Ring-LWE key encapsulation but
relied on conventional polynomial multiplication with high space-delay
products. Cheng et al.~\cite{cheng2021} combined certificateless
schemes with ECC and pseudonyms. \textbf{None of the above amortize
post-quantum costs over a session}---each data exchange requires a
full handshake.

\subsection{Code-Based Encryption}

QC-LDPC code-based key encapsulation provides strong
information-theoretic security foundations. Hu et al.~\cite{hu2019}
analyzed QC-LDPC KEM on FPGA but did not target the IoT low-RAM
profile. Kumari et al.~\cite{kumari2022} combines Ring-LWE and
Diagonal QC-LDPC in a unified scheme validated on MATLAB and Xilinx
Virtex-6 FPGA. We build directly upon their primitives and extend the
data phase with session amortization.

\subsection{Session Management and Amortization}

TLS~1.3 \cite{rfc8446} motivates our design but targets classical
cryptography on high-end hardware. Prior IoT authentication schemes
\cite{reza2021, farash2019} use symmetric session keys but derive them
from classical (ECC) handshakes, providing no post-quantum foundation.
To the best of our knowledge, SAKE-IoT is the \textbf{first scheme to
amortize Ring-LWE + QC-LDPC epoch establishment} over a lightweight
AES-256-GCM data session on constrained IoT hardware with formal
network validation.

\subsection{Ring-LWE}

Ring-LWE is defined over the polynomial ring
$R_q = \mathbb{Z}_q[u]/(u^n+1)$, where $n$ is a power of two and $q$
is prime. Given random $R_n \in R_q^n$ and error
$\varepsilon_n \leftarrow \mathcal{G}^n_\sigma$, the distribution
produces pairs $(R_n, T_n)$ where
$T_n = R_n \delta_n + \varepsilon_n \pmod{q}$ and $\delta_n$ is the
secret key. The hardness of Ring-LWE underpins authentication
security~\cite{lyubashevsky2013}.

\subsection{Ring Signature Scheme}

A ring signature allows a signer to authenticate on behalf of an
\emph{ad-hoc} group of $N$ members without revealing their identity.
We use the Fiat-Shamir ring signature from \cite{kumari2022}, which
employs Bernstein-reconstructed polynomial multiplication.

\subsection{QC-LDPC Key Encapsulation}

A Quasi-Cyclic LDPC code is defined by parity check matrix
$H_{qc} \in \mathbb{F}_2^{X \times Y}$ with circulant structure.
Encapsulation creates syndrome
$CT_0 = [\tilde{W}_l \,|\, I] \cdot \tilde{\varepsilon}^T$ for random
error vector $\tilde{\varepsilon}$; SLDSPA decoding recovers
$\tilde{\varepsilon}$ for session key derivation~\cite{kumari2022}.

\subsection{HKDF-SHA256 and AES-256-GCM}

\textbf{HKDF-SHA256} \cite{rfc5869} provides extract-and-expand key
derivation. For 32-byte output, HKDF-Expand requires a single HMAC
call (RFC~5869~\S2.3).

\textbf{AES-256-GCM} \cite{nist38d} is an authenticated encryption
with associated data (AEAD) scheme providing confidentiality via
AES-256 counter-mode (256-bit key, 14 rounds), integrity via 128-bit
GHASH polynomial authenticator, and AAD binding. The 96-bit nonce
prevents counter reuse when derived as
$\eta = \SID \| ctr$~\cite{nist38d}.

\section{System and Adversary Model}

\subsection{IoT Network Topology}

Our hierarchical IoT network consists of a resource-rich
\emph{Gateway Node} (border router running Contiki-NG) and multiple
\emph{Sensing Nodes} (RFC~7228 Class-1 devices). All communication
uses IEEE~802.15.4 CSMA/CA radio with 6LoWPAN compression and RPL

\subsection{Threat Model}

We adopt the adversary hierarchy of \cite{kumari2022}:
$\mathcal{A}_1$ (passive observer),
$\mathcal{A}_2$ ($\mathcal{A}_1$ + node corruption), and
$\mathcal{A}_3$ ($\mathcal{A}_2$ + oracle access). Security criteria:
\textbf{E1}~Unforgeability, \textbf{E2}~Anonymity,
\textbf{E3}~Unlinkability, and \textbf{E4}~Epoch-Bounded Forward
Secrecy (new in this work).

\section{The SAKE-IoT Protocol}
\label{sec:protocol}

SAKE-IoT operates in four phases, shown in Fig.~\ref{fig:protocol}.

GW $\rightarrow$ Sender: $\{\SID, N_G\}$ (ACK)\\[3pt]

\textbf{Phase 4 --- Epoch Renewal}\\
$\texttt{secure\_zero}(\Kmaster)$; re-initiate Phase~1 }}
\caption{SAKE-IoT protocol overview. Phase~1 is run once per epoch;
Phase~3 repeats for up to $\Nmax$ lightweight data packets.}
\label{fig:protocol}
\end{figure}

\subsection{Phase~1: 1-RTT Post-Quantum Handshake}

The sender performs Ring-LWE key generation, Fiat-Shamir ring
signature generation (N=3), and QC-LDPC syndrome encoding. All
parameters are piggybacked into a single authentication message that
is fragmented into 162 IEEE~802.15.4 frames with ACK-based reliable
delivery. The gateway reassembles, verifies the ring signature,
runs SLDSPA decoding to recover $\tilde{\varepsilon}$, generates
session parameters ($\SID$, nonce $N_G$), and replies with a single
ACK frame. This 1-RTT design eliminates the gateway-to-sender public
key transmission round-trip present in \cite{kumari2022}.

\subsection{Phase~2: Session Key Extraction}

Both parties independently derive the 256-bit master session key:
\begin{equation}
\Kmaster = \text{HKDF-SHA256}(\tilde{\varepsilon} \| N_G,\;
\text{``master-key''}),
\label{eq:kmaster}
\end{equation}
where the gateway nonce $N_G$ provides freshness binding, preventing
Key Compromise Impersonation (KCI) attacks.

\subsection{Phase~3: Amortized Data Transmission}

For each data packet $i$, the sender derives a fresh per-packet key:
\begin{equation}
K_i = \text{HKDF}(\Kmaster,\; \text{``session-key''} \| \SID \| ctr,\;
32),
\label{eq:ki}
\end{equation}
constructs the 96-bit GCM nonce $\eta = \SID \| ctr$, sets
AAD~$= \SID \| ctr$ for IND-CCA2 binding, and encrypts with
AES-256-GCM. The per-packet overhead is exactly
$|\eta| + |\tau| = 96 + 128 = 224$~bits (28~bytes).

\subsection{Phase~4: Epoch Renewal}

When $ctr = \Nmax$, the sender executes
$\texttt{secure\_zero}(\Kmaster, 32)$ and re-initiates Phase~1. The
gateway zeroizes the old session entry before creating the new one,
ensuring Epoch-Bounded Forward Secrecy.

\subsection{Inherited Security}

LR-IoTA \cite{kumari2022}: unforgeability under Ring-LWE hardness
(Theorem~2 of \cite{kumari2022}), anonymity under the Decisional-LWE
assumption, and unlinkability of ring signatures. We prove three
additional properties.

\subsection{Theorem 1: IND-CCA2 Security}

\begin{quote}
\textbf{Theorem~1.}
\label{thm:indcca2}
\textit{Under the standard security of AES-256-GCM (NIST SP~800-38D),
SAKE-IoT Phase~3 is IND-CCA2 secure with advantage bounded by:}
\begin{equation}
\mathrm{Adv}_{\text{IND-CCA2}}(\mathcal{A}) \leq
\mathrm{Adv}_{\text{PRF}}(\text{HMAC}) +
\mathrm{Adv}_{\text{PRP}}(\text{AES}) +
\frac{\Nmax \cdot q_D}{2^{128}}
\label{eq:indcca2}
\end{equation}
\textit{where $q_D$ is the number of decryption oracle queries.}
\end{quote}

\textit{Proof sketch.} AES-256-GCM verifies the 128-bit GHASH tag
\emph{before} any decryption (verify-then-decrypt order). Any tampered
ciphertext receives $\bot$ from the decryption oracle. Since
$K_i = \text{HKDF}(\Kmaster, \SID \| ctr)$ is computationally
indistinguishable from random (HKDF-PRF property, RFC~5869), the
challenge ciphertext is indistinguishable from random noise. The AAD
binding of $\SID \| ctr$ ensures every ciphertext has a unique
cryptographic context. The adversary's advantage reduces to
\eqref{eq:indcca2} $\leq \mathrm{negl}(\lambda)$.

\textit{MATLAB validation:} 500,000 oracle queries (3 adversary
strategies: bit-flip, XOR, random CT); 99.96\% returned $\bot$;
adversary win rate = 0.5037 ($\varepsilon = 0.0037$, negligible at
$\lambda = 128$). MAC-before-decrypt tested with real Java
\texttt{javax.crypto.Mac} HMAC-SHA256: 10,000/10,000 tampered CTs
rejected (forgery probability $\leq 2^{-256}$). \hfill$\square$

\subsection{Theorem 2: Replay Resistance}

\begin{quote}
\textbf{Theorem~2.}
\label{thm:replay}
\textit{For any packet with $ctr \leq \mathit{last\_seq}$:}
\begin{equation}
\Pr[\text{Receiver accepts replayed packet}] = 0.
\label{eq:replay}
\end{equation}
\end{quote}

\textit{Proof.} The gateway enforces $ctr > \mathit{last\_seq}$
\emph{before} any key derivation or decryption. Since legitimate
transmissions advance $\mathit{last\_seq}$ monotonically, every
replayed packet is rejected with probability \emph{exactly} zero---a
deterministic guarantee, not a probabilistic one.

\textit{Corollary (Desynchronization safety):} Under packet loss, the
counter self-heals to the nonce of the last received packet. Skipped
nonces are implicitly expired; replays of dropped nonces are rejected
since their $ctr \leq \mathit{last\_seq}$.

\textit{MATLAB validation:} 10,000 replay attempts: 0 accepted
(Pr~=~0, exact); 10,000 valid sequential: 10,000 accepted; 20\% drop
rate: all 8,033 received packets accepted, counter self-healed;
10,000 duplicates: 0 accepted. \hfill$\square$

\subsection{Theorem 3: Epoch-Bounded Forward Secrecy}

\begin{quote}
\textbf{Theorem~3.}
\label{thm:ebfs}
\textit{Let $\Kmaster^{(k)}$ and $\Kmaster^{(k+1)}$ be master keys of
consecutive epochs. After epoch~$k$ terminates and
$\Kmaster^{(k)}$ is zeroized:}

\textit{(1) Past Secrecy:}
$\Pr[\mathcal{A}(\Kmaster^{(k+1)}) \text{ recovers } K_i^{(k)}]
\leq \mathrm{negl}(\lambda)$.

\textit{(2) Future Secrecy:}
$\Pr[\mathcal{A}(\Kmaster^{(k)}) \text{ predicts } K_i^{(k+1)}]
\leq \mathrm{negl}(\lambda)$.
\end{quote}

\textit{Proof.} Past secrecy: recovering $\Kmaster^{(k)}$ from
$\Kmaster^{(k+1)}$ requires inverting HMAC-SHA256 (PRF) or solving a
new Ring-LWE instance to recover $\tilde{\varepsilon}^{(k)}$
(Eq.~23, Thm.~2 of \cite{kumari2022})---both computationally
infeasible. Future secrecy: $\Kmaster^{(k+1)}$ is derived from a
fresh, independently sampled error vector
$\tilde{\varepsilon}^{(k+1)}$; computing
$\tilde{\varepsilon}^{(k+1)}$ from $\Kmaster^{(k)}$ requires
solving a new Ring-LWE instance. Epoch bounds
($\Nmax = 2^{20}$, $\Tmax = 86{,}400$~s) guarantee bounded
adversary exposure.

\textit{MATLAB validation:} MS zeroization confirmed (zero-vector
post-overwrite); 0/100 past keys recoverable; 0/100 future keys
predictable; 5 epochs $\times$ 10 keys fully mutually isolated
(2,000 cross-epoch pairs). \hfill$\square$

\subsection{MATLAB Theoretical Evaluation}

\subsubsection{Metric M1: Computational Latency}

The per-packet Tier~2 cost is measured as 0.068~ms
(HKDF-SHA256~$\approx$~0.019~ms + AES-256-GCM~$\approx$~0.047~ms,
with $\times$1.20 GHASH overhead factor) versus the base paper's
7.3728~ms per-packet HE cost (Table~7, \cite{kumari2022}):

\begin{equation}
\text{Reduction} = \frac{7.3728 - 0.068}{7.3728} = \mathbf{99.1\%}.
\label{eq:latency}
\end{equation}

Break-even occurs at $N{=}4$ packets. The epoch initiation cost
of 22.55~ms is a one-time overhead amortized across up to
$\Nmax = 2^{20}$ packets.

\subsubsection{Metric M2: Per-Packet Bandwidth}

The base paper transmits $CT_0 = 408$~bits (51~bytes) of QC-LDPC
syndrome per packet. SAKE-IoT replaces this with:

\begin{equation}
|\eta| + |\tau| = 96 + 128 = 224 \text{ bits (28 bytes)}.
\label{eq:bw}
\end{equation}

\begin{equation}
\text{BW Saving} = \frac{408 - 224}{408} = \mathbf{45.1\%}.
\label{eq:bwsave}
\end{equation}

This saving is deterministic and N-independent---it applies to every
Tier~2 packet. At $N{=}1{,}000$, the cumulative saving is
$184{,}000$~bits ($\approx$22.5~KB).

\subsection{Contiki-NG/Cooja Network Simulation}

\subsubsection{Experimental Setup}

Two Cooja Mote (JVM) nodes simulate a sender and gateway over an
IEEE~802.15.4 virtual radio channel. The Contiki-NG OS stack includes
CSMA/CA, 6LoWPAN, and RPL Lite routing. A custom JavaScript metrics

until two complete amortization cycles (25 data messages, 2 renewals)
complete.

\subsubsection{Metric N1: Authentication E2E Delay}

The authentication phase transmits 10,317~bytes over 162 6LoWPAN
fragments with ACK-based reliable delivery:

\begin{equation}
\Delta_{\text{auth}} = 8{,}242.68 \text{ ms.}
\label{eq:authdelay}
\end{equation}

This is dominated by 6LoWPAN fragmentation overhead, not
computation---confirming that amortization is critical from a
\emph{network} perspective.

\subsubsection{Metric N2: Data Phase E2E Latency}

\begin{equation}
\Delta_{\text{data}} = 25.096 \text{ ms.}
\label{eq:datadelay}
\end{equation}

\begin{equation}
\text{E2E Reduction} = \frac{8{,}242.68 - 25.096}{8{,}242.68}
= \mathbf{99.7\%}.
\label{eq:e2ered}
\end{equation}

This exceeds the 99.1\% MATLAB compute-only figure because the
single-frame data packet (52~bytes UDP) costs drastically less
than the 162-fragment auth payload.

\subsubsection{Metric N6: Amortization Efficiency Ratio}

Let $B_{\text{auth}} = 10{,}317$\,B, $b = 29$\,B/msg. The AER for
$N$ messages per epoch:

\begin{equation}
\text{AER}(N) = \frac{(N-1) \cdot B_{\text{auth}}}
{N \cdot (B_{\text{auth}} + b)}.
\label{eq:aer}
\end{equation}

At $N{=}20$: $\text{AER}(20) = \mathbf{94.7\%}$. The unamortized
protocol requires $20 \times 10{,}317 = 206{,}340$\,B vs.\ our
$10{,}317 + 20 \times 29 = 10{,}897$\,B---confirmed directly from
Cooja packet counts.

%% VIII. DISCUSSION
%% ======================================================================
\section{Discussion}
\label{sec:discussion}

\textbf{Cooja JVM and CPU timing.} Cooja Motes execute C code on a
JVM, absorbing CPU computation into the host processor. The reported
E2E latencies (8,242~ms auth, 25~ms data) represent pure network
transit times (6LoWPAN fragmentation, CSMA/CA backoff, RPL routing).
On real MSP430 hardware at 8~MHz, AES-256-GCM computation adds
$\approx$0.99~ms per packet---less than 4\% of the 25~ms network
latency, thus negligible.

\textbf{N\_max scaling.} The simulation uses $\Nmax{=}20$ for
practical runtime; deployable systems use $\Nmax{=}2^{20}$. The
amortization mechanism is independent of the threshold value.

\textbf{AES-256-GCM vs. AES-128-CTR/HMAC-SHA256.} We chose
AES-256-GCM over AES-128-CTR + HMAC-SHA256 for three reasons:
(1)~256-bit AES key provides 128-bit post-quantum security via
Grover's bound; (2)~GCM's 28-byte overhead (vs.\ 36~bytes for
CTR+HMAC) exactly matches the MATLAB bandwidth model; and (3)~GCM is
a single-pass AEAD, reducing implementation complexity.

\textbf{RAM footprint.} The AES-256-GCM crypto stack requires
$\approx$450~bytes of peak stack. The Ring-LWE polynomial arrays
($n{=}512$) require $\approx$30.9~KB for the sender. This exceeds
Class-1 device limits (8~KB) but is suitable for Cooja Motes and
Class-2+ devices. Deployment on Class-1 hardware requires reduced
parameters ($n{=}32$).

%% ======================================================================
%% IX. CONCLUSION
%% ======================================================================
\section{Conclusion}
\label{sec:conclusion}

This paper presented SAKE-IoT, a four-phase session amortization
protocol that bridges the efficiency gap between post-quantum
cryptographic strength and IoT operational feasibility. By extending
Ring-LWE ring signature authentication and QC-LDPC code-based key
encapsulation with a 1-RTT piggybacked handshake, HKDF-SHA256
session key derivation, and AES-256-GCM authenticated encryption,
SAKE-IoT achieves three independently validated performance
improvements: 99.7\% E2E latency reduction, 45.1\% per-packet
bandwidth saving, and 94.7\% session-level bandwidth saving at
$N{=}20$. Three formal security proofs (IND-CCA2, replay
resistance, Epoch-Bounded Forward Secrecy) cover the session layer,
and a complete Contiki-NG/Cooja network emulation over IEEE~802.15.4
provides the first physical-layer validation of a post-quantum
session amortization protocol, with 100\% packet delivery ratio and
two verified epoch renewal cycles.

\textbf{Future work:} (1)~Multi-node simulations with 3--5 senders.
(2)~Time-based epoch expiry ($\Tmax$). (3)~Port to real Z1 hardware
(MSP430) using $n{=}32$ Ring-LWE parameters.
(4)~Energest radio and CPU energy profiling.
